P3-CPU-设计文档

发表于2025-10-29|更新于2025-10-31|CO
|浏览量:

设计要求

  • 32位单周期处理器
  • 指令集
    1. add(无符号加法，不考虑溢出)
    2. sub(无符号减法，不考虑溢出)

    3. ori
    4. lw

    5. sw
    6. beq
    7. lui
    8. nop(机器码为0x00000000，不进行任何有效行为)
  • 模块化与层次化设计
    • 顶层有效的驱动信号包括且仅包括异步复位信号reset
    • clk采用内置时钟模块

整体结构

设计思路:
“上游”:实际上是Moore型FSM

  • 输入下一个pc值并存在 PC寄存器中[状态存储模块]
  • 下一个周期 PC 寄存器将其中的 pc 值传给 ROM
  • ROM 取得指令并输出给 splitter[输出电路]

“下游”:实际上是Mealy型FSM

  • 从 splitter 输入指令的各段二进制码
  • 由 Controller 生成控制信号
  • 寄存器堆根据指令进行读写、改变其中存储的数据，输出下一个 pc值[状态转移电路NPC]

模块定义

数据通路

IFU(取指令单元)

这里包含了PC和IM两个模块.

PC(程序计数器)

  • 寄存器实现
  • 异步复位功能,复位值为起始地址(0x000030000)
  • 地址范围:0x00003000 ~ 0x00006FFF

    考虑到异步复位是到0x0000000,我们选择在输出PC_OUT前将PC值加上0x00003000,从而保证复位.

IM(指令存储器)

  • ROM实现
    • 容量: 4096 × 32bit
    • 内部起始地址为0

      $4096=2^12$,ROM实际地址宽度仅需12位.考虑到ROM是按字(4位)存储,因此ROM_Addr = (PC<<4)[11:0]，实际就是取PC[13：2]

      1. 端口定义

      |信号名|方向|位宽|描述|
      |:——:|:——:|:——:|:—-:|
      |CLK|I|1|时钟信号|
      |RST|I|1|异步复位信号|
      |Next_PC|I|32|下一条要被执行的指令的ROM地址|
      |Instr|O|32|输出当前正在进行的指令|
      |PC_OUT|O|32|当前正在执行的指令的地址|
      |PC|O|32|当前正在执行指令的ROM地址|

2. 功能定义

功能名称 功能描述
写PC寄存器 在时钟上升沿到来时,将下一条指令的地址(Next_PC)写入寄存器
取指令 根据当前的PC值从IM中读出对应的指令到Instr端口
复位 当Reset信号有效时PC寄存器中的值置为0x00000000

NPC(下一指令计算单元)

1. 有关控制信号

我们知道NPC可能会有两种状态:

  • 非跳转指令:PC = PC + 4
  • 跳转指令beq:PC = (PC+4)+(Ext_sign(imm16)<<2)
    由此我们需要一个选择信号判断是否跳转.
    进一步,只有Rs==Rt时跳转,因此branch = beq && equal.

2. 端口定义

信号名 方向 位宽 描述
CLK I 1 时钟信号
ZERO I 1 是否达到beq跳转条件
Branch I 1 当前是否是跳转指令
imm16 I 16 目标label转化的16位立即数
PC I 32 当前指令的ROM地址
Next_PC O 32 下一条要被执行的指令的ROM地址

Splitter

分割得到二进制码段.
R_format(add,sub):

  • Opcode(6),Rs(5),Rt(5),Rt(5),Rd(5),shamt(5),func(6)
    其它指令(sw,lw,ori,lui,beq):
  • pcode(6),Rs(5),Rt(5),imm16(16)

EXT(位扩展器)

可以使用 Logisim 内置的Bit Extender,将16位立即数扩展到32位.操作较为简单,没有单独封装为子电路.

有关控制信号

imm16 -> Ext_OP

  • 0 : zero_ext(ori)
  • 1 : sign_ext(lui,lw,sw)

GRF(通用寄存器组)

  • 32个具有写使能的寄存器
  • 有异步复位功能
  • 0号寄存器始终为0,其它初始值为0

1. 端口定义

信号名 方向 位宽 描述
CLK I 1 时钟信号
RST I 1 异步复位信号
A1 I 5 地址输入信号，指定32个寄存器中的一个，将其中的数据读出到RD1
A2 I 5 地址输入信号，指定32个寄存器中的一个，将其中的数据读出到RD2
A3 I 5 地址输入信号，指定32个寄存器中的一个，将其作为写入目标
WD I 32 数据输入信号
WE I 1 写使能信号
RD1 O 32 输出A1指定的寄存器中的32位数据
RD2 O 32 输出A2指定的寄存器中的32位数据

2. 功能定义

功能名称 功能描述
复位 Reset信号有效时，所有寄存器中储存的值均被清零
读数据 读出A1，A2地址对应的寄存器中储存的数据，将其加载到RD1和RD2
写数据 当WE信号有效且时钟上升沿来临时，将WD中的数据写入到A3地址对应的寄存器

3. 有关控制信号

  • A1 <- Rs
  • A2 <- Rt
  • A3 <- Reg_Dst(add,sub)
    • 0 : Rt(lw,ori,lui)
    • 1 : Rd(add,sub)
  • WE <- Reg_Write(add,sub,lw,ori,lui指令时有效)
  • WD <- MemtoReg(Lw指令有效)
    • 0 : ALU_Result (add,sub,ori,lui)
    • 1 : DM_Read_Data(lw)

ALU(算数逻辑单元)

  • 提供 32 位加(add)、减(sub)、或运算(ori)及大小比较(beq)功能。
  • 加减法按无符号处理

1. 端口定义

信号名 方向 位宽 描述
A I 32 参与ALU运算的第一个值
B I 32 参与ALU运算的第二个值
ALU_ctr I 2 ALU功能选择信号
zero O 1 相等判断信号
Result O 32 输出ALU运算结果

2. 功能定义

功能名称 ALU_ctr 功能描述
移位 00 Result = B << 16
或 01 Result = A B
加 10 Result = A + B
减 11 Result = A - B

3. 有关控制信号

  • A <- RS
  • B <- ALU_src
    • 0 : Rt(add,sub,beq)
    • 1 : imm16(lui,lw,sw,ori)
  • zero -> 控制信号ZERO(beq)
  • Result -> Mem_to_Reg
    • 0 : GRF_WD(add,sub,lui,ori)
    • 1 : DM_A(lw,sw)

DM(数据储存器)

  • RAM实现,双端口模式,设置属性Separate load and store ports
  • 容量为 3072 × 32bit (与IM类似12位取地址)
  • 异步复位功能
  • 复位值:0x00000000
  • 地址范围:0x00000000 ~ 0x00002FFF
  • 起始地址:0x00000000

1. 端口定义

信号名 方向 位宽 描述
A I 32 地址输入信号,指向数据储存器中某个存储单元
WD I 32 数据输入信号
WE I 1 写使能信号
ld I 1 读使能信号
CLK I 1 时钟信号
RST I 1 异步复位信号
RD O 32 输出A指定的存储单元中的32位数据

2. 功能定义

功能名称 功能描述
读数据 读出A地址对应的存储单元中的数据,将其加载到RD
写数据 当WE信号有效且时钟上升沿来临时,将WD中的数据写入到A地址对应的存储单元
复位 Reset信号有效时,所有寄存器中储存的值均被清零

3. 有关控制信号

  • A <- ALU_Result
  • Data <- GRF_Read_Data2
  • WD <- MemWrite(sw指令有效)
  • ld <- MemtoReg(lw指令有效)

控制器

Controller

设计思路:和电路+或电路
根据上面的控制信号需求,我们整合设计出如下控制信号输出模块.

1. 端口定义

信号名 方向 位宽 描述
OP_code I 6 指令类型
Reg_Dst O 1 数据输入信号
MemtoReg O 1 DM读使能信号
Reg_Write O 1 GRF中WD写入数据选择信号
MemWrite O 1 DM写入使能信号
Branch O 1 跳转使能信号
Ext_Op O 1 扩展类型信号
ALUop<2:0> O 2 ALU指令类型

2. 真值表

R_format ori lui lw sw beq
OP_code 1 0 0 0 X X
Reg_Dst 0 1 1 1 1 0
MemtoReg 0 0 0 1 X X
Reg_Write 1 1 1 1 0 0
MemWrite 0 0 0 0 1 0
Branch 0 0 0 0 0 1
Ext_Op X 0 0 1 1 X
ALUop<2:0> X OR SHIFT ADD ADD SUB

ALU_ctr

Controller只是实现了对Op_code的解析,而R型指令的前6位全为0,因此增加了ALU_ctr来实现对Func的解析.

1. 端口定义

信号名 方向 位宽 描述
Func I 6 R_format指令类型
ALUop O 2 Ctrl的ALUop信号
ALUctr O 2 最终的ALU运算类型控制信号

2. 对应关系

  • Func:
    • add -> ADD
    • sub -> SUB
  • ALUop 直接对应即可

思考题

上面我们介绍了通过 FSM 理解单周期 CPU 的基本方法。请大家指出单周期 CPU 所用到的模块中，哪些发挥状态存储功能，哪些发挥状态转移功能。

状态存储:PC寄存器,GRF通用寄存器,IM指令存储器,DM数据存储器.
状态转移:NPC模块,以及数据通路的其它部分.

现在我们的模块中 IM 使用 ROM， DM 使用 RAM， GRF 使用 Register，这种做法合理吗？ 请给出分析，若有改进意见也请一并给出。

合理.
ROM是只读存储器，是非易失的，可以用来储存指令；
RAM可以读写,足DM对读写的要求；
GRF是寄存器堆,需要较高的读写速度,因此适合用寄存器实现.

在上述提示的模块之外，你是否在实际实现时设计了其他的模块？如果是的话，请给出介绍和设计的思路。

我增加了Splitter和ALU_ctr模块,在上文已经有过介绍.

事实上，实现 nop 空指令，我们并不需要将它加入控制信号真值表，为什么？

nop指令码为0x00000000,相当于sll $0, $0, 0.
也就是把$0寄存器中的值左移0位并写入$0寄存器,因为$0的值始终为0,不会被修改,因此该指令执行后没有任何影响.
同时nop指令不会造成任何操作,对电路不造成影响.

阅读 Pre 的 “MIPS 指令集及汇编语言” 一节中给出的测试样例，评价其强度（可从各个指令的覆盖情况，单一指令各种行为的覆盖情况等方面分析），并指出具体的不足之处。

指令的覆盖情况: 没有覆盖sub指令.
单一指令的覆盖行为:

  • 计算类指令:
    • 没有覆盖边界数.
    • 没有覆盖目标寄存器为$0的情况.
  • 存取类指令:
    • 没有覆盖offset为负数的情况.
    • 对于lw没有覆盖目标寄存器为$0的情况.
    • 对于sw没有做到存入的word中每个byte都不是零.
  • 跳转类指令:
    没有覆盖一下情况:
    • 跳转，且目标在此跳转指令之前.
    • 跳转，且目标是此跳转指令.
    • 不跳转，且目标在此跳转指令之前.
    • 不跳转，且目标是此跳转指令.
